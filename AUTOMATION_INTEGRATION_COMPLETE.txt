HOW IT WORKS
================================================================================

WORKFLOW:
1. Create task file in AI_Employee_Vault/Needs_Action/
2. Orchestrator detects file (polls every 30 seconds)
3. Reads frontmatter: type=linkedin_automation / whatsapp_automation / gmail_automation
4. Routes to appropriate processor
5. Processor extracts task details from Markdown
6. Calls MCP Server wrapper
7. MCP Server calls automation module
8. Automation module (Playwright/IMAP) performs action
9. Result logged with log_action()
10. File moved to Done/ on success or Quarantine on failure

EXAMPLE TASK FILE:
---
type: linkedin_automation
task_type: post
source: linkedin
status: pending
---

## Post Content
🚀 Excited to announce something amazing!

## Image Path
/path/to/image.jpg

Result: File moves to Done/ after successful post

================================================================================
TROUBLESHOOTING
================================================================================

Q: "LinkedIn not authenticated"
A: Check LINKEDIN_EMAIL and LINKEDIN_PASSWORD are set

Q: "Session expired"
A: Delete ~/.linkedin_session/state.json and re-login

Q: "WhatsApp QR not appearing"
A: Kill chromium processes: pkill -f chromium

Q: "Gmail not authenticated"
A: Verify App Password (16 chars, from Google Account)

Q: "Contact not found in WhatsApp"
A: Use exact contact name as it appears in WhatsApp

Q: "Media file not found"
A: Use absolute paths, not relative paths

Q: "Task not processing"
A: Check file is in Needs_Action/ with correct frontmatter
